# 部署须知 + 近期变更汇总

> 给部署用的速查页。完整后端步骤见 `backend/README.md`，完整历史见 `docs/CHANGELOG.md`。

## 一、架构回顾

```
浏览器 → GitHub Pages（前端 index.html，公开源码）
           ↓ 登录拿 token，带 Bearer 调 API
阿里云 nginx(HTTPS) → Flask(backend/app.py)
           ↓ 读/写
backend/banks/*.json   ← 题库（每套一个文件，已 .gitignore，只 demo 进仓库）
backend/users.json     ← 账号密码（已 .gitignore）
backend/secret.key     ← token 签名密钥（自动生成，已 .gitignore）
```

## 二、近期几个版本改了什么

| 版本 | 变更 |
|---|---|
| 前后端分离 | 题库从 `data/` 搬到服务器 `backend/banks/`，manifest 后端自动扫描生成；前端加登录闸门；新增「管理题库」页（登录后网页里增删题库）；保留离线下载 |
| 安全加固 | token 改 HMAC 签名（防伪造）；登录失败限流；账号/题库/密钥全部移出公开仓库；CORS 限定 Pages 域名 |
| 背题模式 | 题库卡新增「背题」：整页列出全部题 + 答案，解析默认折叠、可一键展开全部 |
| 显示大小 | 顶栏「Aa」按钮，全局缩放所有界面（紧凑/标准/宽松或 −/＋），记忆在本地 |
| 自动演示模式 | 预览/本地自动用演示账号读本地 demo；线上 Pages 自动走真实后端；无需手改 `DEMO_MODE`（`?demo=1/0` 可强制） |
| 去敏 | 真实账号密码、服务器地址全部用占位符；公开源码仓库不含敏感信息 |
| 乱序模式 | 答题页（练习/考试通用）「乱序」开关，随机打乱题序，答案不丢、状态持久化 |

## 三、部署清单（每次发布对照）

### 服务器（阿里云，题库/账号改了才动）
1. 账号：编辑 `backend/users.json`（已 .gitignore，设强密码）→ `systemctl restart <服务名>`
2. 题库：上传 json 到 `backend/banks/`，或登录后在「管理题库」页导入——**即时生效，与 git 无关**
3. 后端代码 `app.py` 改了：同步到服务器 → `systemctl restart`
4. 确认：HTTPS 正常、安全组只放行 80/443/22、`ALLOWED_ORIGIN` = 你的 Pages 地址

### 前端（GitHub Pages，UI/功能改了才发）
1. **替换占位符**：把 `index.html` 里的 `YOUR_BACKEND_HOST` 换成真实后端主机
   （例：`sed -i 's#YOUR_BACKEND_HOST#你的IP.sslip.io#' index.html`）——**这一步别漏，否则线上连不上后端**
2. 确认 `DEMO_MODE = false`（保持，预览/本地会自动切演示）
3. `git push` → Pages 1–2 分钟生效

## 四、三条最容易踩的坑

1. **发布前没替换 `YOUR_BACKEND_HOST`** → 线上「Failed to fetch」。替换后再 push。
2. **真实账号密码 / 服务器地址别提交公开仓库**：仓库里只能是占位符；真实值只在服务器 / 发布时注入。
   （注：Pages 发布后那份前端里的后端地址，浏览器仍能看到，这是 Pages 固有特性；安全靠登录鉴权 + CORS + HTTPS。）
3. **题库变更不走 git**：直接在服务器 `banks/` 或网页「管理题库」操作即时生效；git push 只更新前端 UI。

## 五、登录与账号

- 真实账号在服务器 `backend/users.json`；改完重启服务生效。
- 想让所有登录立刻失效：删服务器 `backend/secret.key` 再重启（重签密钥，全员需重新登录）。
- 演示账号 `student / exam2026` 仅在演示模式（预览/本地）有效，不连后端、无风险。
